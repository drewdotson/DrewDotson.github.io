Projects

Knoxville Utilities Board Spreadsheet Automation

During my time as an intern for Knoxville Utilities Board's fiber department. I was assigned the task of creating analytical automation script for updating the contents of a major Excel spreadsheet that most of the department uses to enter and view data regarding KUB fiber installation. The updates include moving certain completed jobs to an archive sheet and creating / modyfing sheets used specifically for data analysis. You can view the script code and output here.

To complete this task, I followed the agile development model which included breaking the assignment into three separate phases and delivering working software for review upon completion of each phase. I used Python as the main tool to complete this assignment (heavily utilizing the Pandas and OpenpyXL libraries). Excel itself is only utilized to retrieve the file and save its changes. All of the heavy lifting of this task is done by the Python script.

Fiber Internet Company Customer Tracking System

In one of my undergrad courses, I was assigned a project of creating a database application using Python with embedded SQLite for interacting with the database file. I chose to create a customer tracking system for an imaginary fiber company. The application, while built in Python has most of its interaction heavily funneled through SQLite queries using the SQLite3 library. You can view the application code and interact with the application here. The code includes a sample database to work with.

The application was created by closely following the modular programming software design technique. Each different type of software functionality is contained within its own file. Multiple types of checks and constraints are put into place to ensure data integrity.

Video Game Review Scores Dashboards

In one of my undergrad courses, I was assigned a project of creating visualizations in Tableau using a dataset of my choice. I chose to create two dashboards showing many related metrics of video game review scores. The dataset was pulled from Kaggle. You can view the dashboards here.

The dashboards are identical to each other besides the fact that one depicts player review scores and one depicts critic review scores. Multiple views exist to depict many metrics of the review scores such as game vs. game, developer vs. developer, average scores per genre, etc. The dashboards also contain a multitude of filtering options to allow users to specify what metrics to analyze.
